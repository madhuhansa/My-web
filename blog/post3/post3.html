Before you can analyze or model your data, you need to clean it. Dirty data can lead to wrong insights and poor predictions. In this post, we’ll walk through the basics of data cleaning and what to look out for

1. Identifying Weird Values

Look for values that do not make logical sense in their columns. 

# Find unexpected values in a numeric column
df[df['age'] > 100]  # Ages over 100
df[df['price'] < 0]   # Negative prices

# Check categorical columns for weird entries
df['category'].value_counts()

Common issues include placeholder values, test entries, or corrupted data.
You can use smart coding techniqes without spending too much time on manual inspection.
Check for categorical values that should not exist, like "N/A", "unknown", or "test".

2. Handling Whitespace

Invisible whitespace can break string matching: (e.g., " Apple" instead of "Apple")

#look for repeting objects because of whitespace
df['colname'].unique()  # Check unique values
        
# Strip whitespace from string columns
df['text_column'] = df['text_column'].str.strip()

Whitespace can be tricky, especially in string columns. It can make new values appear different from existing ones, even if they look the same visually. that is why you should always check unique for each object column.

3. Fixing Data Type Mismatches

Ensure each column has the correct data type:

# Check current dtypes
df.dtypes

# Convert types
df['date_column'] = pd.to_datetime(df['date_column'])
df['numeric_str'] = pd.to_numeric(df['numeric_str'], errors='coerce')

# Handle currency columns
df['price'] = df['price'].replace('[\$,]', '', regex=True).astype(float)

Data types are crucial for analysis. If a column is supposed to be numeric but is stored as a string, it can lead to errors in calculations. Always check the data types of your columns and convert them as necessary. also whitespace can change the data type.

4. Fixing Units mismatches

Ensure each column has the correct data type:

# Check current dtypes
df.dtypes

# Standardize units
df['Height_m'] = df['Height_cm'] / 100

Units can vary widely in datasets, especially in scientific or engineering contexts. Always standardize units to a common format before analysis. For example, if you have height in both centimeters and meters, convert everything to meters.

5. Fixing Typos

Typos can lead to duplicate entries or misclassifications:

# Fix typos in categorical columns
df['Category'] = df['Category'].replace({'Elecronics': 'Electronics', 'electonics': 'Electronics'})

Typos can create duplicates or misclassifications in your data. Always check for common misspellings and correct them. You can use string matching techniques to identify and fix typos in categorical columns.
